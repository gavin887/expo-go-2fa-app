# Google 身份验证器类 App - 功能大纲

这是一个使用 Expo go 进行开发的 类 Google 身份验证器类 的 App，支持 TOTP、HOTP 两种主流算法。

## 一、账号管理

### 1.1 添加账号
- **扫码添加**
  - 调用相机扫描 OTP 二维码
  - 自动解析 `otpauth://` 格式 URL
- **手动输入**
  - 表单字段：Issuer（发行方）、Account（账号）、Secret（Base32 密钥）
  - 实时预览当前验证码（帮助用户确认密钥正确性）
- **粘贴板导入**
  - 读取系统剪贴板内容
  - 自动识别以下格式：
    - 完整 `otpauth://` URL
    - 纯 Base32 Secret（如 `JBSWY3DPEHPK3PXP`）
    - `Issuer:Secret` 格式
  - 识别成功后跳转预览确认页
  - 识别失败时提示支持的格式并引导手动输入

### 1.2 查看账号列表
- 每个账号卡片显示：
  - Issuer + Account 名称
  - 当前 6 位动态验证码（大号等宽字体）
  - 30 秒倒计时进度（圆环或进度条）
- 空状态提示：引导用户添加第一个账号

### 1.3 编辑账号
- 修改名称（Issuer / Account）
- 修改 Secret 密钥（需二次确认，提示会影响验证码生成）
- 实时预览修改后的验证码

### 1.4 删除账号
- 左滑卡片显示删除按钮
- 或长按卡片弹出删除菜单
- 删除前需二次确认

---

## 二、验证码显示与交互

### 2.1 自动刷新
- 每隔 30 秒自动重新计算所有账号的验证码
- 倒计时从 30 秒递减至 0 秒时重置为 30 秒并刷新验证码

### 2.2 手动交互
- **点击验证码**：复制到剪贴板，并显示“已复制”提示（Toast）
- **点击卡片其他区域**：可选行为（如展开详情，默认不做额外跳转）

### 2.3 倒计时视觉反馈
- 剩余时间 < 5 秒时，倒计时圆环/进度条变为红色
- 其他时间使用主题色（如绿色/蓝色）

---

## 三、安全与存储

### 3.1 本地加密存储
- 所有账号数据（Secret、名称、自定义排序等）加密存储
- 加密密钥由设备安全环境托管（如 Keychain / Keystore）

### 3.2 应用锁（可选扩展）
- 启动时要求输入 PIN 码
- 或使用生物识别（指纹/面容）解锁
- 可开关，默认关闭

---

## 四、辅助功能

### 4.1 导出账号（纯文本格式）
- 支持两种导出格式，用户二选一：
  - **Secret 列表**：每行一个 Base32 Secret
  - **otpauth 链接列表**：每行一个完整的 `otpauth://` URL
- 导出后调用系统分享面板（保存到文件、发送到其他 App）

### 4.2 备份账号（加密 JSON）
- 将所有账号数据（包括名称、排序、自定义字段）打包为加密 JSON 文件
- 用于完整恢复账户状态
- 调用系统分享面板保存文件

### 4.3 导入账号
- 支持两种导入来源：
  - **从备份文件导入**（加密 JSON）
    - 选项：覆盖现有数据 / 合并数据（去重）
  - **从导出文件导入**（纯文本格式）
    - 每行解析为 Secret 或 otpauth 链接
    - 批量添加账号，遇错跳过并报告
- 调用系统文件选择器选择文件

### 4.4 时间校准
- **原理**：向公共时间服务器获取标准时间，计算与本地时间的差值
- **生命周期**：
  - App 启动时自动校准一次
  - 从后台切回前台时可选择性重新校准
- **差值应用**：生成验证码时使用 `本地时间 + 缓存差值` 作为当前时间戳
- **用户控制**：
  - 开关：启用/禁用时间校准（默认启用）
  - 手动校准按钮
  - 显示当前差值（如“服务器时间比本地快 0.32 秒”）
  - 显示上次校准时间

### 4.5 主题切换
- 浅色模式 / 深色模式 / 跟随系统
- 实时切换，无需重启 App

## 五、UI 结构

### 5.1 页面列表

| 页面 | 说明 |
|------|------|
| 账号列表（主页） | 展示所有账号卡片，底部浮动添加按钮 |
| 添加账号（动作菜单） | 底部弹出三个选项：扫码 / 手动 / 剪贴板导入 |
| 扫码页面 | 全屏相机预览，自动解析二维码 |
| 手动输入表单页 | 填写 Issuer、Account、Secret，实时预览验证码 |
| 剪贴板导入预览页 | 展示解析结果，确认后保存 |
| 编辑账号页 | 修改名称或 Secret，支持删除账号 |
| 设置页 | 分组列表：外观、安全、数据管理、高级、关于 |

### 5.2 设置页详细结构

#### 分组 A：外观
- 主题切换（浅色 / 深色 / 跟随系统）

#### 分组 B：安全
- 应用锁开关（PIN / 生物识别）

#### 分组 C：数据管理
- 导出账号（弹出格式选择：Secret 列表 / otpauth 列表）
- 备份账号（生成加密 JSON）
- 导入账号（弹出来源选择：备份文件 / 导出文件）

#### 分组 D：高级
- 时间校准开关（默认开启）
- 当前差值显示
- 上次校准时间显示
- 立即校准按钮

#### 分组 E：关于
- 版本号
- 开源协议 / 隐私政策

### 5.3 通用组件

- **账号卡片**：圆角 12–16，轻微阴影，主题自适应背景
- **验证码字体**：等宽（monospace），粗体，28–32sp
- **倒计时指示器**：圆环或进度条，动态填充，颜色随剩余时间变化（绿 → 橙 → 红）
- **交互反馈**：Toast 提示，按钮按压态，卡片轻触动画
- **空状态**：居中插画 + 引导文案

## 七、用户典型操作流程

### 流程 1：首次添加账号
1. 打开 App → 空状态 → 点击 ⊕
2. 选择“从剪贴板导入”
3. 系统读取剪贴板 → 自动识别 → 预览确认页
4. 点击保存 → 返回账号列表 → 新卡片出现
5. 验证码开始倒计时，点击验证码 → 复制成功

### 流程 2：批量备份与恢复
1. 进入设置 → 数据管理 → 备份账号
2. 生成加密 JSON → 保存到本地或云端
3. 换设备或重置 App → 进入导入账号
4. 选择“从备份文件导入” → 选择文件 → 恢复完成

### 流程 3：使用时间校准
1. 进入设置 → 高级 → 开启时间校准
2. App 自动获取服务器时间 → 显示差值
3. 生成验证码时自动应用时间差
4. 如怀疑偏移，点击“立即校准”手动更新

## 八、技术约束说明（Expo Go 环境）

- 所有功能基于纯 JavaScript 和 Expo SDK 可实现
- 不使用需要预编译的自定义原生模块
- 加密存储使用 `expo-secure-store`
- 二维码扫描使用 `expo-barcode-scanner`
- 剪贴板读写使用 `expo-clipboard`
- 文件选择使用 `expo-document-picker`
- 时间校准通过 `ntp-client` 实现


```ntp-client-usage
ntpClient.getNetworkTime("0.cn.pool.ntp.org", 123)
  .then((date) => {
    console.log("本地时间：", new Date());
    console.log("网络时间：", date);

    // 获取本地时区偏移
    const localOffset = new Date().getTimezoneOffset() * 60000;

    // 获取网络时区偏移
    const networkOffset = date.getTimezoneOffset() * 60000;

    // 计算时差
    const offsetDiff = localOffset - networkOffset;

    // 同步本地时间
    const synchronizedTime = new Date(date.getTime() + offsetDiff);
    console.log("同步后的本地时间：", synchronizedTime);
  })
  .catch((err) => {
    console.error(err);
  });
```
